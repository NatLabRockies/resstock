\paragraph{Description}
% \input{characteristics/description/Bedrooms}
The number of bedrooms in the housing unit.

\paragraph{Distribution Data Sources}
% \input{characteristics/sources/Bedrooms}
\begin{itemize}
    \item 2017 and 2019 American Housing Survey microdata.
    \item Building type categorization based on U.S.~EIA 2009 RECS.
\end{itemize}

\paragraph{Direct Conditional Dependencies}
\input{characteristics/dependencies/Bedrooms}

\paragraph{Options}
The options for the Bedrooms characteristic are integers from 1 to 5; see Table \ref{table:hc_opt_bedrooms}. The number of bathrooms is always defaulted based on the number of bedrooms.

\input{characteristics/options/Bedrooms}

Property definitions are shown in Table \ref{table:hc_arg_def_bedrooms}. See the OpenStudio-HPXML \href{https://openstudio-hpxml.readthedocs.io/en/v1.10.0/workflow_inputs.html#hpxml-building-construction}{Building Construction} documentation for additional information. The number of bedrooms is used many places in the OpenStudio-HPXML model workflow.

\input{characteristics/properties/Bedrooms}

\paragraph{Distribution Assumptions}
% \input{characteristics/assumptions/Bedrooms}
\begin{itemize}
    \item More than 5 bedrooms are labeled as 5 bedrooms and 0 bedrooms are labeled as 1 bedroom.
    \item Limit 0--499 ft\textsuperscript{2} housing units to only 1 or 2 bedrooms. The geometry measure has a limit of (ffa-120)/70 >= bedrooms.
\end{itemize}
